\note{
 	Das Navigieren durch Elemente ist eines der ältesten Probleme der Informatik.
 	Gelöst wurde es immer mit Arrays und Schleifen, wobei der Programmierer dann
 	auf Dinge achten musste wie Grenzen u.ä.
}

\begin{frame}[fragile]{Lösung in modernen Programmiersprachen}
	\begin{exampleblock}{StandardIteratorBeispiel.java}
        \begin{lstlisting}
List<String> zauberspruch = new ArrayList<>();
zauberspruch.add("Abra");
zauberspruch.add("kada");
zauberspruch.add("bra");

for (String myString: zauberspruch)  {
    System.out.print(myString);
}
        \end{lstlisting}
 	\end{exampleblock}
\end{frame}

\note{
	Moderne Programmiersprachen bieten den Iterator als einheitliche Schnittstelle
	für unterschiedliche Datenstrukturen an. Die for-each-Schleife funktioniert für 
	alle Klassen, die Iterable implementieren. 
}

\frameWithImageOnly{\BILDERPFAD/problem3.jpg}
\note{
	Da stellt sich die Frage: Warum sollte man dann selbst das Muster noch implementieren, wenn
	die Programmiersprache bereits eine Implementierung mitbringt?
}
