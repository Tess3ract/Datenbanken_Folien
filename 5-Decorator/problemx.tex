\note{
	Wir möchten gerne eine Spiel entwickeln, in dem der Spieler Gebäude bauen und erweitern kann. Die Erweiterungen verbessern die Produktion und die Verteidigung der Gebäude. Nachts können die Gebäude des Spielers von Monstern angegriffen werden. Monster haben eine Grundstärke, die mithilfe von Rüstungsgegenständen und Waffen verbessert werden kann. 
}

\note{
	Folgender Vorschlag wurde unterbreitet: Wir erweitern die Klasse um boolsche Attributen für
	jeden Ausrüstungsgegenstand und die Methode calculateDamage() berechnet aus der Belegung dieser Attribute den Schaden.
	Die Unterklasse verwenden dann in ihrer Implementierung von calculateDamage() die Methode der
	Oberklasse. 
}
\begin{frame}{Probleme}
	\begin{itemize}
 		\item Wartungsalptraum
 		\item Unübersichtliche calculateDamage()-Methode
 		\item Schwer zu erweitern
 		\item Verstößt gegen Open-Closed-Prinzip
 	\end{itemize}
\end{frame}
